// build a reactive graph in code > const g = new Graph("pricing"); > const price = g.state("price", 100); > const tax = g.derived("tax", [price], ([p]) => p * 0.1); > const total = g.derived("total", [price, tax], ([p, t]) => p + t); // ask the graph to describe itself > g.describe(); { nodes: [price, tax, total], edges: [(price→tax), (price→total), (tax→total)] } // ask the graph WHY > g.describe({ explain: { from: "price", to: "total" } }); { path: ["price", "tax", "total"], branches: [{ via: "tax", transform: "× 0.1" }, { direct: true }], result: 110 } // — every change has a causal chain. nothing is hidden.
section / 03 / a small vocabulary

four words
you'll use forever.

state /steɪt/
noun · mutable cell
A writable value, no dependencies. The leaf input of the graph. Push a value into it; downstream propagates.
producer /prəˈdjuːsər/
noun · push source
Wraps a timer, event stream, or async iterator into the graph. Emits values as a reactive source, never polled.
derived /dɪˈraɪvd/
verb · pure function of deps
Recomputes when its dependencies change. Diamond-safe: fan-out and fan-in produce a single recomputation.
effect /ɪˈfɛkt/
noun · side-effect sink
Where the graph touches the world. Logs, fetches, DOM writes, file IO — always at the leaf, never in the middle.
