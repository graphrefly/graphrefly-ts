FOUR LAWS THE GRAPH OBEYS.

01

Code is the source. Spec is the projection.

Factories carry their own description through factoryTag. graph.describe() projects the live topology as a JSON blueprint. The map is never out of date.

PROJECTION
02

Every value has a causal chain.

graph.describe({ explain: { from, to } }) walks backwards through dependencies and returns a structured chain. Humans read it. Agents parse it. No black boxes.

AUDIT
03

Diamond resolution is the contract.

Fan-out, fan-in, single recomputation. The protocol gates DATA on RESOLVED, so downstream sees one coherent update — not two, not partial. Glitch-free by construction.

CORRECT
04

Subgraphs have owners. Owners have terms.

L0 static, L1 TTL, L2 heartbeat, L3 supervisor. The ownership staircase lets multiple agents edit a shared topology without colliding. validateOwnership enforces it on every PR.
