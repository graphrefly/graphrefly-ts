The Protocol

From Elegant Simplicity
to a Full Reactive Graph

It began with callbag—a protocol of arresting simplicity where every source, sink, and operator shared one function signature. That economy of form was a revelation: composable streams with no framework, no classes, no registries. But as systems grew in ambition—as AI agents demanded inspectable state, as human operators needed audit trails, as orchestration crossed language boundaries—the single-function signature reached its limits.

GraphReFly inherits callbag’s core insight—that a protocol should be minimal, composable, and free of hidden magic—but reimagines it for a world where graphs are artifacts, not just execution traces. Every node is addressable. Every message is a tuple—the same shape in TypeScript and Python—reviewable, replayable, LLM-friendly. The two-phase push model (DIRTY, then DATA) ensures diamond-safe propagation without glitches.

The result is a protocol designed equally for human eyes and machine reasoning. State is observable without disturbing data flow. Graphs can be serialized, diffed, and versioned. What was once an ephemeral stream becomes a durable, inspectable structure—a shared workspace where humans and language models co-operate with full transparency.

“One protocol. Messages are arrays of tuples. Same shape for TS and Python—reviewable, replayable, LLM-friendly.”
Protocol Message Format
// Messages are tuples:
[Type, Data?]

// Type enum:
START  = 0
DATA   = 1
END    = 2
DIRTY  = 3

// Two-phase push:
// 1. DIRTY propagates
// 2. DATA resolves
// = diamond-safe
Principles

One Protocol

Messages are tuples. Nodes are functions. No classes, no framework, no hidden registries. The entire protocol fits on an index card—yet composes into systems of arbitrary complexity.

Diamond-Safe

Two-phase push (DIRTY then DATA) eliminates glitch propagation. Nodes that share upstream dependencies converge correctly without manual scheduling or topological sorts.

Graph as Artifact

Graphs are serializable, diffable, and versionable. State is observable without disturbing data flow. The reactive graph is not just execution—it is a persistent, inspectable structure.

Human + LLM

Designed for co-operation between human operators and language models. Audit trails, natural-language descriptions, and full observability make the graph a shared workspace for both.

Example
Listing 1 — Basic Reactive Graph TypeScript 
const count = state(0);
const doubled = derived([count], (c) => c * 2);

effect([doubled], (d) => console.log(d));

count.down([[DATA, 1]]);  // logs: 2
Applications
Orchestration

Coordinate complex workflows across nodes. Define dependencies declaratively and let the graph resolve execution order through two-phase propagation.

Message Queue

Reliable, observable message passing with built-in backpressure. Every message is a tuple—inspectable, replayable, and language-agnostic.

AI Agentic Memory

Inspectable state for AI agents. The graph serves as persistent, structured memory that both humans and LLMs can query and reason about.

Security & Audit

Audit trails built into the protocol. Every state change is traceable. Observability is a first-class concern, not an afterthought.

Human + AI Operations

Co-designed systems where human operators and language models share the same reactive graph—full transparency, no hidden state.
