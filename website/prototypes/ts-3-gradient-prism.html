Why GraphReFly

Three problems, one protocol.

Information overload, opaque AI, and boilerplate glue code. GraphReFly solves all three with a single reactive graph.

Stop Drowning in Information

Thousands of emails, alerts, feeds, and messages hit you every day. GraphReFly is a universal reduction engine: it ingests chaotic streams and outputs only what matters. Reactive nodes filter, aggregate, and prioritize automatically — no polling, no cron jobs, no glue scripts.

Trust AI by Understanding It

Every node in the graph is observable. Every derivation is a causal chain you can inspect, replay, and audit. Checkpoints persist state; traces show exactly why the AI made each decision. You don't have to blindly trust a black box — you can verify it.

Describe It, Don't Code It

Write what you want in plain English. An LLM composes the reactive graph for you. The graph runs persistently, checkpoints automatically, and you can inspect or override any node at any time. No boilerplate. No framework lock-in. Zero dependencies.

How It Works

Three steps. No magic.

From a natural-language description to a running, traceable, persistent automation.

Describe

Tell the system what you want in plain language: "Triage my inbox — flag urgent items, summarize the rest into a daily digest, archive noise." That's your spec.

LLM Composes Graph

An LLM reads your description and emits a reactive graph: source nodes for email ingestion, derived nodes for classification and summarization, effect nodes for notifications. All wired together declaratively.

Runs Reactively

The graph runs persistently. State checkpoints automatically. Every derivation is traced. You can inspect, override, or extend any node — no restart required. Reactive propagation, not polling.

Code Preview

Feels like state management.
Works like a reactive engine.

Familiar primitives. Unfamiliar power.

counter.ts 
import { Graph } from '@graphrefly/graphrefly';
import { state, derived, effect } from '@graphrefly/graphrefly/core';
import { DATA } from '@graphrefly/graphrefly/protocol';

// 1. Create a reactive graph
const g = new Graph();

// 2. State node — holds a value, pushes updates
const count = g.add(state(0));

// 3. Derived node — recomputes when count changes
const doubled = g.add(derived(
  [count],
  (c) => c * 2
));

// 4. Effect node — side-effect on every change
g.add(effect(
  [doubled],
  (d) => console.log(`doubled: ${d}`)
));

// 5. Push an update — entire graph reacts
count.down([[DATA, 1]]);
// logs: doubled: 2
