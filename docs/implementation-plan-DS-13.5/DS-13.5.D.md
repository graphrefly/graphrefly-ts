## Locked decisions

Read [docs/implementation-plan.md, "DS-13.5.D"](../implementation-plan.md) for canonical lock text.

### D.2 — Keep `tryIncrementBounded`, no new primitive

**Decision:** Original `boundedCounter` proposal would have wrapped existing helper into a bundle whose contents (count Node already exists, `isAtCap` is a one-line derived, `tryIncrement` IS the existing function, reset is one line) add zero new semantics. Per L2 §44 / `feedback_no_imperative_wrap_as_primitive`: **keep the imperative helper**.

**Optional (deferred):** widen `cap` parameter from `number` to `number | Node<number>` to support reactive caps without wrapping. One-line branch reading `.cache` if Node-shaped. Land this only when a consumer surfaces.

**Reingest-topology declared-deps rewrite:** parked indefinitely. No consumer signal; harness works imperatively today; Phase 14 doesn't depend on it.

### D.3 — `routeJobIds` collision contract

**Decision:** JSDoc + regression test only. No code change.

- Document on [src/patterns/harness/types.ts:79–87](../../src/patterns/harness/types.ts:79) (`IntakeItem.relatedTo` field): items lacking `relatedTo[0]` and producing colliding `trackingKey`s overwrite the prior entry; retry/reingest items must set `relatedTo[0]` to the original key.
- Add a regression-pinning test asserting last-write-wins behavior for the documented contract.
- **Hidden invariant:** ack runs before reingest publishes (harness flow invariant — preserved).
- Framework-enforced uniqueness rejected (would break legitimate retry/reingest patterns).

### D.4 — Per-claim eval-verifier subgraph mounting

**Decision:** Mount per-claim eval subgraph at `eval/${claimId}` on the JobFlow's graph. Cleanup on claim ack/nack.

- Same per-call-subgraph-with-cleanup shape as DS-13.5.C alternative A.
- Candidate for shared `perCallSubgraph(parent, prefix, factory)` helper extraction once both DS-13.5.C and D.4 land.
- After fix: `describe()` shows `eval/<claimId>::{candidates, dataset, scoresNode, raw, filter}` while claim is active; cleared after ack/nack.
