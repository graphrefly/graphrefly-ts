# DS-13.5.A — INVALIDATE protocol redesign (Bundle 1.5)

**Status:** LOCKED Q1–Q16 (2026-05-01). Implementation-ready, no design questions remain.

**Scope summary:** Promote `INVALIDATE` from tier-2 to its own tier-4 settle group between value-settle (tier-3 DATA/RESOLVED) and terminal (tier-5/6 COMPLETE/ERROR). Resolves Agent 5 deadlock pattern where INVALIDATE alone left dependents DIRTY without settling. Cross-repo: spec + TLA+ + library code.

**Cross-repo touches:** YES — this DS edits `~/src/graphrefly/GRAPHREFLY-SPEC.md` and `~/src/graphrefly/formal/wave_protocol.tla` in addition to `graphrefly-ts/src/`. Commit them in their respective repos.

## Locked decisions (Q1–Q16)

Read the canonical lock table inline at [docs/implementation-plan.md, "DS-13.5.A — INVALIDATE protocol redesign"](../implementation-plan.md). The Q-table holds the decision text verbatim. **Do not paraphrase or re-interpret it.** Notable summary:

| Q | Decision summary |
|---|---|
| Q1 — DATA + INVALIDATE same wave | DATA wins; source's `_cached` becomes new DATA value |
| Q2 — Cascade scope | Source's `_cached` clears; downstream `prevData[source-slot]` resets to `undefined`; downstream's own `_cached` untouched |
| Q3 — RESOLVED + INVALIDATE same wave | RESOLVED wins |
| Q5 — `data[i]` shape | Widen to `T[] | null | undefined`: `undefined` = no message, `null` = INVALIDATE, `[]` = RESOLVED, `[v, ...]` = DATA |
| Q6 — INVALIDATE + COMPLETE/ERROR | Compose naturally via tier ordering; INVALIDATE first, then terminal |
| Q7 — INVALIDATE + PAUSE/RESUME | Buffer alongside DATA; replay on RESUME in arrival order |
| Q8 — INVALIDATE + equals | Not a value emission; never substituted; always propagates |
| Q9 — INVALIDATE inside `batch()` | Order-independent merge: DATA wins (Q1), RESOLVED wins over INVALIDATE (Q3), INVALIDATE+INVALIDATE collapses |
| Q10 — INVALIDATE + replayBuffer | INVALIDATE clears replay buffer too; late subscribers post-INVALIDATE see SENTINEL state |
| Q11 — INVALIDATE + resubscribable | INVALIDATE does NOT trigger producer resubscription |
| Q12 — Auto-cascade default | Reading A: only when a dep INVALIDATED in the wave AND fn doesn't emit / returns `undefined | null`. New gate opt `invalidateWhenDepsInvalidate?: boolean` (default `true`) |
| Q13 — `NodeFnCleanup.invalidate?` hook | Already shipped Package 6 (2026-04-23); no change |
| Q14 — TLA+ + fast-check | Required: `Invalidate` action settles wave; new `InvalidateSettlesWave` invariant; `MergeRulesRespected` invariant; `EqualsFaithful` extension; fast-check single-INVALIDATE-settles + order-independent-merge properties |
| Q15 — Spec amendment scope | §1.3 (tiers): INVALIDATE as tier-4. §1.3.1 (auto-DIRTY-prefix): extend. §2.4: rewrite. §2.5: cross-ref Q10. §2.7: clarify `data[i] === null` as INVALIDATE marker. New §2.4a: merge rules table. New §5.13: `invalidateWhenDepsInvalidate` opt + cascade contract |
| Q16 — TEARDOWN sequencing | Framework auto-emits COMPLETE (or ERROR) BEFORE propagating TEARDOWN. Spec §2.6 amendment: "auto-precede with COMPLETE/ERROR" rule. Cascade implementation in `Graph._destroyClearOnly` and `Graph.remove` walks subtree, emits synthetic terminal. Removes manual workaround at `processManager.dispose()` line 1292 |

**Additional ungated decisions (lean-locked):**
- `withStatus`: stays in current status on INVALIDATE; defer `invalidated?: number` companion until consumer demand.
- `attachStorage` / WAL: persist INVALIDATE for replay determinism; spec amendment to §8.7.
- Worker bridge wire protocol: fold into DS-14 (bridge protocol changes anyway under changesets/delta work).
- Codec envelope (`Graph.snapshot()`): no change.

- **Rule 1.1** — DIRTY precedes DATA/RESOLVED; tier-3 mutually exclusive.
- **Rule 1.5** — Auto-DIRTY-prefix synthesis when tier-3 present and node not already dirty. **Q15 extends this to tier-4 INVALIDATE.**
- **Rule 1.7** — PAUSE/RESUME tier-3 buffering. **Q7 extends this to INVALIDATE.**

- **Rule 1.10** — Meta TEARDOWN fan-out fires at top of `_emit` before parent's state-transition walk. **Q16 amends: COMPLETE/ERROR auto-prefix before TEARDOWN.**
- **Rule G.20-cleanup-default** — Cleanup `() => void` fires on INVALIDATE in addition to deactivation/pre-re-run.
- **Rule P.41** — Tier-3 wave exclusivity; Q9 maintains analog at tier-4 (INVALIDATE+INVALIDATE collapses).
- **Rule M.4** — `undefined` is global SENTINEL; `T | null` only valid DATA. **Q5 widens `data[i]` shape; the new `null` sentinel is NOT user-emittable DATA, it's a wave-presence marker for INVALIDATE.**
- **Rule M.21** — `ctx.prevData[i] === undefined` is canonical "never emitted DATA" detector. Q5 makes `prevData[i] === null` orthogonal: "INVALIDATE happened in this wave."

## Watch-outs (hidden invariants from Q6)

- **INVARIANT:** INVALIDATE settles a wave (decrements `_dirtyDepCount`). Without this, the deadlock pattern returns. Counter-test in fast-check (`single-invalidate-settles`) is the regression sentinel.
- **INVARIANT:** `data[i] === null` is reserved as the "INVALIDATE arrived this wave" marker, orthogonal to `prevData[i] === undefined` (SENTINEL). Library code must treat `null` and `undefined` distinctly for `data[i]`.
- **INVARIANT:** Cascade default for `invalidateWhenDepsInvalidate` is `true`. New nodes opting out must set `false` explicitly. Symmetric with `errorWhenDepsError` / `completeWhenDepsComplete`.
- **INVARIANT:** Sugar fn returning `[]` (explicit zero-emit) is NOT an auto-INVALIDATE trigger; only `undefined | null` returns are (Q12). Don't change today's RESOLVED-on-empty behavior.
- **INVARIANT:** INVALIDATE under `pausable: "resumeAll"` buffers alongside DATA, replays in arrival order on final-lock RESUME (Q7). Equals substitution applies during replay (Rule 1.7).
- **INVARIANT:** TEARDOWN cascade now auto-precedes with COMPLETE/ERROR (Q16). The synthetic terminal must NOT fire if the node is already in terminal state (idempotency).
- **INVARIANT:** Codec envelope (`Graph.snapshot()`) writes whatever's in `_cached` at snapshot time; INVALIDATE-then-snapshot writes a SENTINEL slot naturally (no codec change needed).
