## COMPOSITION-GUIDE pointers

- **Rule G.7 (feedback cycles)** — proposed effect reads `store.entries`, writes `entryCreatedAtNs`. Distinct nodes — no self-feedback. Clean.
- **Rule P.5 (factory wiring order)** — wire effect AFTER `entryCreatedAtNs` reactive map exists; mount as child node on the parent graph.
- **Rule M.21 (`prevData[i] === undefined` SENTINEL)** — effect's idempotency check uses `entryCreatedAtNs.has(key)` to skip already-set entries. Not the same as SENTINEL semantics; this is reactive-map state, not graph SENTINEL.
- **Rule L2.35 (imperative-controller-with-audit)** — `entryCreatedAtNs.set` from inside the effect is a sanctioned mutation method on a reactiveMap (Rule G.4 / G.6 family).
- **Rule M.4** — `undefined` SENTINEL semantics: when `score` runs before effect, `entryCreatedAtNs.get(key)` returns `undefined`; existing `?? nowNs` fallback handles this gracefully.

// After permanentKeys + entryCreatedAtNs reactive maps are constructed (~line 322)
// and after store.entries is wired,
const syncEffect = effect(
  [storeEntriesNode],  // exact dep — confirm with code
  (entries: ReadonlyMap<string, MemoryEntry<TMem>>) => {
    const nowNs = monotonicNs();
    for (const key of entries.keys()) {
      if (!entryCreatedAtNs.has(key)) {
        entryCreatedAtNs.set(key, nowNs);
      }
    }
  },
);
this.add(syncEffect, { name: "entryCreatedAtNs/sync" });
this.addDisposer(keepalive(syncEffect));

## Watch-outs (hidden invariants from Q6)

- **INVARIANT:** Effect runs before first archival scan, OR `score` fn handles undefined-timestamp via `?? nowNs` fallback (already the case today). Do not remove the fallback — it's the race-window mitigation.
- **INVARIANT:** Effect is idempotent — re-emissions of `store.entries` for already-tracked keys skip via `entryCreatedAtNs.has(key)` check before write.
- **INVARIANT:** `entryCreatedAtNs` GC cleanup ([memory-composers.ts:427](../../src/patterns/ai/memory/memory-composers.ts:427)) still drives via the existing subscriber that compares against active keys. The new sync effect adds keys; the existing GC removes them. Both paths must remain.
- **INVARIANT:** Snapshot portability preserved — same keys, same timestamps; only the write path changes. Snapshot codec round-trip should produce identical observable state pre- and post-fix.
- **INVARIANT:** `retention.score` is pure post-fix — no mutation traceable from inside. Audit by code review: search `retention.score` body for any `.set(`, `.delete(`, `.add(` after the fix.
- **INVARIANT:** No §7 feedback cycle introduced. Effect reads `store.entries`, writes `entryCreatedAtNs`. Distinct nodes. Confirm via `graph.describe()` walk.
- **DO NOT WIRE** `retention.onArchive`. Park as POST-1.0 carry; current distill retention path handles archival. Don't add the callback hook now without a consumer.
