## Locked decisions

Read [docs/implementation-plan.md, "DS-13.5.E"](../implementation-plan.md) for canonical lock text.

**Per-site discriminator** via `kind` field. Matches `ProcessInstance` precedent ([src/patterns/process/index.ts:122–142](../../src/patterns/process/index.ts:122)). `audit` field stays **optional** at all mutation sites; caller opts in by passing `audit: { append, kind, build }` to `lightMutation` opts.

**4 record types + 4 keyOf exports:**

```ts
export interface TopicPublishRecord extends BaseAuditRecord {
  kind: "topic.publish";
  topicName: string;
  itemKey: string;  // result of topic.keyOf(item)
}
export const topicPublishKeyOf = (r: TopicPublishRecord): string =>
  `${r.topicName}::${r.itemKey}`;

export interface SubscriptionAckRecord extends BaseAuditRecord {
  kind: "subscription.ack";
  subscriptionId: string;
  cursor: number;
}
export const subscriptionAckKeyOf = (r: SubscriptionAckRecord): string =>
  `${r.subscriptionId}::${r.cursor}`;

export interface SubscriptionPullAndAckRecord extends BaseAuditRecord {
  kind: "subscription.pullAndAck";
  subscriptionId: string;
  cursor: number;
  itemCount: number;
}
export const subscriptionPullAndAckKeyOf = (r: SubscriptionPullAndAckRecord): string =>
  `${r.subscriptionId}::${r.cursor}`;

export interface HubRemoveTopicRecord extends BaseAuditRecord {
  kind: "hub.removeTopic";
  topicName: string;
}
export const hubRemoveTopicKeyOf = (r: HubRemoveTopicRecord): string =>
  r.topicName;
```

**`HubAddTopicRecord` deferred** until Hub.addTopic surfaces an audit consumer (no current callsite emits this record).
