# DS-13.5.B — Tier 5.2 reactive-options widening (5 primitives + companions)

**Status:** FULLY LOCKED 2026-05-01.

**Scope summary:** Widen 5 resilience primitives (`timeout`, `budgetGate`, `retry`, `circuitBreaker`, `rateLimiter`) to accept reactive-option Nodes with **state preservation across rebind**. Replace today's switchMap-rebuild-state-loss pattern. Pre-1.0 break for `retry` / `circuitBreaker` (current "reset on opts swap" semantics goes away). Bundle returns `<Primitive>Bundle<T>` with companion `<Primitive>State` Node.

**Cross-cutting** (apply to all 5 primitives):

- **(α) State preservation across rebind.** Rebind = parameter tweak only, never state reset. Mode-axis transitions throw on the opts Node.
- **`StatusValue` central enum widened** to `"pending" | "running" | "completed" | "errored" | "cancelled" | "paused"` per the StatusValue follow-on lock. Used as literal vocabulary inside lifecycle-shaped `<Primitive>State`.
- **`GateState` central enum** ships at `src/extra/resilience/gate-state.ts` as `"open" | "closed"`. Used as literal vocabulary inside gate-shaped `<Primitive>State`. `rateLimiter` extends with `"throttled"`; `circuitBreaker` extends with `"half-open"`.
- **`Node<Partial<Options>>` on emit; primitive merges with prior options.** Empty `{}` is a no-op.
- **Construction-time validation:** throw if first opts emit is missing/invalid required fields.
- **`processManager` migrates** `status: "compensated"` → `status: "cancelled"` (separate small change, but coupled to the StatusValue lock).

### timeout (canary, fully sub-locked)

**File:** [src/extra/resilience/timeout.ts](../../src/extra/resilience/timeout.ts) (or wherever `timeout` lives — find under `src/extra/resilience/`).

```ts
import type { StatusValue } from "../resilience/status.js";

export type TimeoutOptions = {
  ns: number;
  meta?: Record<string, unknown>;
};

export type TimeoutState =
  | { status: "pending" }
  | { status: "running"; startedAt_ns: number; deadline_ns: number }
  | { status: "completed"; settledAt_ns: number }
  | { status: "errored"; firedAt_ns: number; deadline_ns: number };

export type TimeoutBundle<T> = {
  node: Node<T>;
  timeoutState: Node<TimeoutState>;
};

export function timeout<T>(
  source: Node<T>,
  opts: Partial<TimeoutOptions> | Node<Partial<TimeoutOptions>>,
  extraOpts?: ExtraOpts,
): TimeoutBundle<T>;
```

| Axis | Policy | In-flight reconciliation |
|---|---|---|
| `ns` change | Rebind | In-flight deadline NOT changed; new `ns` applies to next attempt only |
| `meta` change | Rebind | Updates immediately; merged onto next attempt's emitted node meta |
| Empty `{}` emit | No-op | Treated as "nothing changed"; no rebind, no companion fire |

**Construction-time validation:** throw if first opts settle missing/non-positive `ns`. `Node<Partial<TimeoutOptions>>` with `cache === undefined` pauses source until first valid opts emit.

**Companion semantics:** `timeoutState` is snapshot-shaped; default `Object.is`-on-status-field equals. Subscribers don't re-fire on identical-shape transitions but DO fire on every state transition AND on payload changes within the same status. **`timeoutState` covers the `lastTimeout`-style use case** via `status === "errored"` payload (carrying `firedAt_ns` + `deadline_ns`). No separate event-shaped companion.

### Pipeline forwarding contract

**File:** [src/extra/resilience/resilient-pipeline.ts](../../src/extra/resilience/resilient-pipeline.ts:236).

For each primitive: if `opts.<primitive>` is **static** `Partial<Options>`, build today's static path (switchMap rebuild remains for non-yet-widened primitives). If **Node-form**, forward the Node directly to the widened primitive (no switchMap rebuild). Companion `<primitive>State` lifted onto the pipeline bundle when caller passes Node-form. ## Watch-outs (hidden invariants from Q6)

- **INVARIANT:** Mode-axis change throws on the opts Node (not on the call to the primitive). Caller wraps mode-change-via-switchMap-rebuild as the recipe.
- **INVARIANT:** `Node<Partial<Options>>` with `cache === undefined` pauses source until first valid opts emit (B3 (d) gate pattern from earlier triage).
- **INVARIANT:** Construction-time validation throws if first opts settle is missing required fields. Caller errors fail loud at wire time, not at first source DATA.
- **INVARIANT:** `<Primitive>State` is snapshot-shaped; equals on status field by default. Caller can override `equals` if they need different dedup.
- **INVARIANT:** `timeoutState` covers the `lastTimeout`-style consumer via `status === "errored"` payload. No separate event-shaped companion.
- **INVARIANT:** `circuitBreaker.now` is constructor-only (clock override is structural). Throw on opts emit if `now` field appears.
- **INVARIANT:** `rateLimiter.maxBuffer` mode-locked at construction. Throw if opts emit changes Infinity ↔ bounded.
- **INVARIANT:** Pipeline forwarding skips switchMap rebuild only for widened primitives. As more primitives widen, more pipeline branches skip the rebuild.
