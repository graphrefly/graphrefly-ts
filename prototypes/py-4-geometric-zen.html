Why GraphReFly

Three problems. One protocol.

01
Stop Drowning in Information
Information arrives faster than you can reduce it. GraphReFly turns every source into a reactive node that filters, derives, and prioritizes — so you see only what matters.
02
Trust AI by Understanding It
Every node carries its lineage: which inputs, which derivation, when. Decisions become auditable causal chains, not black-box suggestions you take on faith.
03
Describe It, Don't Code It
State what you need in plain language. An LLM composes the reactive graph. You review, edit, and run it. No framework lock-in. No hidden complexity.

Applications

Scenarios

  • ✉
    Email Triage
    Classify, prioritize, and route incoming messages reactively. High-urgency surfaces instantly; newsletters batch for weekly review. Every routing decision is traceable.
  • ⚠
    Spending Alerts
    Transactions flow through derived nodes that track category totals, detect anomalies, and alert on threshold breaches. No polling — pure reactive propagation.
  • ★
    Knowledge Management
    Notes, bookmarks, and research feed into a graph that clusters by topic, surfaces connections, and produces weekly digests. Your second brain, with receipts.
Quick Start

The code

demo.py
from graphrefly import state, derived, effect, DATA # 1. Create a stateful node count = state(0) # 2. Derive a computed value doubled = derived([count], lambda c: c * 2) # 3. React to changes effect([doubled], lambda d: print(d)) # 4. Push data — prints: 2 count.down([(DATA, 1)])

State. Derivation. Effect. Data flows through messages — no callbacks, no polling, no imperative triggers.
