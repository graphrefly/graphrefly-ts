GraphReFly: A Reactive Graph Engine
for Human + LLM Co-operation

Abstract.   Describe automations in plain language. Review them visually. Run them persistently. Trace every decision. GraphReFly turns information overload into reactive, auditable workflows that humans and LLMs co-author.

$ pip install graphrefly
Section 1

Problem Statement

Modern knowledge workers and developers face a common crisis: the volume of information, decisions, and automated actions outpaces any individual's capacity to manage them. Current tooling either demands extensive code or produces opaque AI outputs that erode trust.

1.1 Stop Drowning in Information

Your inbox holds 147 unread messages. Three Slack channels need responses. A dashboard shows 12 alerts. By the time you've triaged one source, two more have escalated. The problem isn't laziness — it's that information arrives faster than any human can reduce it.

GraphReFly treats every information source as a reactive node in a graph. Nodes derive, filter, and prioritize automatically. You see only what matters, the moment it matters.

Key insight: the bottleneck is reduction, not access. We have too many feeds, not too few.

1.2 Trust AI by Understanding It

LLMs are powerful but opaque. When an AI assistant says "you should cancel this subscription," why? Which data led to that? Without causal traceability, every AI recommendation requires a leap of faith.

Every node in a GraphReFly graph carries its lineage: which inputs produced which outputs, when, and through what derivation logic. Decisions become auditable chains, not black-box suggestions.

"If you can't trace the reasoning, you can't trust the result."
cf. explainable AI (XAI) literature — GraphReFly provides structural traceability rather than post-hoc rationalization.

1.3 Describe It, Don't Code It

Building an automation today means choosing a framework, writing boilerplate, wiring event handlers, managing state. Most people with the domain knowledge to design workflows lack the engineering time to implement them.

GraphReFly inverts the model: describe what you want in natural language, and an LLM composes the reactive graph. The graph is inspectable, editable, and runnable — no framework lock-in, no hidden complexity.

1 Natural language descriptions are compiled to graph topology by an LLM composer. The resulting graph can be inspected and hand-edited before execution.

Section 3

Applications

GraphReFly is domain-agnostic, but these scenarios illustrate the reduction-engine pattern across personal and professional use.

✉
Email Triage
Incoming messages are classified, prioritized, and routed reactively. High-urgency items surface instantly; newsletters batch for weekly review. Every routing decision is traceable.
⚠
Spending Alerts
Transactions flow through derived nodes that track category totals, detect anomalies, and alert when thresholds are breached. No polling — pure reactive propagation.
📚
Knowledge Management
Notes, bookmarks, and research feed into a graph that clusters by topic, surfaces connections, and produces weekly digests. Your second brain, with receipts.

Section 4

Quick Start

Install the Python package and define your first reactive graph in under ten lines:

from graphrefly import state, derived, effect, DATA # 1. Create a stateful node count = state(0) # 2. Derive a computed value doubled = derived([count], lambda c: c * 2) # 3. React to changes effect([doubled], lambda d: print(d)) # 4. Push data — prints: 2 count.down([(DATA, 1)])
Fig. 2: Minimal reactive graph — state, derivation, effect.

The state node holds a value. derived recomputes when dependencies change. effect observes and acts. Data flows through messages — no callbacks, no polling, no imperative triggers.

The tuple (DATA, 1) is a message in GraphReFly's protocol. Type DATA = 1 signals a value update.
Section 5

Implementations

GraphReFly is a protocol, not a language. Two reference implementations are available, sharing the same spec and message semantics.

Python
graphrefly
Zero-dependency reactive graph for Python 3.10+. Async-ready. LLM-composable. Persistent checkpoints.
pip install graphrefly
TypeScript
@graphrefly/graphrefly-ts
TypeScript-first reactive graph. Tree-shakable. NestJS adapter. Full message protocol.
npm i graphrefly-ts

Both implementations conform to the GraphReFly Spec, ensuring portable graph definitions and interoperable tooling.
