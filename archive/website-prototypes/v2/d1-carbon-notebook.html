Chapter 00 · Field Notes from a Reactive Graph

Code is the topography.
Spec is the map.
Agents read both.

Premise

The library projects your code into a structured topology — humans diff it, agents reason over it, both edit the same graph without colliding.

— graphrefly/spec §1.0
Method

One primitive — node — composed into state, producer, derived, effect. Every value is reactive; every change has a causal chain.

— graphrefly/spec §2.1
01 On the difference between a state container and a graph.

See also: COMPOSITION-GUIDE §4, "lazy activation".

A state container forgets. A reactive graph remembers why.

Most libraries store values and notify subscribers. GraphReFly stores values together with the dependency edges that produced them — so when a downstream value changes, you can walk the graph backwards and ask: which inputs caused this, and which transforms ran on the way? graph.describe({ explain: { from, to } }) returns a structured causal chain.

The graph is the data. Not a representation of the data — the data itself. describe() is the map; observe() is the magnifier; the live values flowing through the topology are the territory. Humans and agents read the same structure.

This is what the spec calls projection. You do not maintain a separate blueprint document and pray it matches your runtime. The runtime is the blueprint, projected on demand.

Specimen — minimal graph
// counter, with derivation and explanation
const g = new Graph("counter");
const count   = g.state("count", 0);
const doubled = g.derived(
  "doubled", [count],
  ([c]) => c * 2
);

g.effect([doubled], ([d]) =>
  console.log("doubled:", d)
);

// → doubled: 0
count.down([[DATA, 3]]);
// → doubled: 6

// — and the trace —
g.describe({ explain:
  { from: "count", to: "doubled" } });
Specimen — explain output
{
  "path": ["count", "doubled"],
  "trigger": { tier: "DATA", value: 3 },
  "recomputed": true,
  "version": 4
}
Notes from the field

Three problems that state doesn't solve.

i. Black-box agents

Why did the agent decide that?

Most agent frameworks return an answer. GraphReFly returns the answer plus the causal chain that produced it — every dependency, every transform, every guard. Auditable by construction.

ii. Spec drift

The blueprint never matches the code.

Hand-maintained YAML specs lag the code that implements them. GraphReFly inverts the relationship: code is the source, spec is the projection. The map is always current.

iii. Multi-agent collision

Two agents, one shared topology.

Subgraph ownership through a structural protocol — L0 static, L1 TTL, L2 heartbeat, L3 supervisor. Concurrent edits across worktrees don't corrupt the shared graph.

— "The graph is its own documentation, its own audit log, and its own coordination protocol."
