A Field Guide

Four species compose every graph.

You will encounter them in every habitat — a chat agent, a pricing model, a knowledge base, a control loop. Learn to recognize them and the rest reveals itself.

Status quietus

— state · the dormant cell

A writable seed. Holds a value, propagates when pushed. The leaf input of any graph. No dependencies, no behavior, no surprise.

familia: leafg.state()

Producens vivax

— producer · the source

A push source. Wraps a timer, event stream, async iterator, websocket. Emits values reactively — never polled, never busy-waiting.

familia: sourceg.producer()

Derivata cogitans

— derived · the thinker

A pure function of upstream dependencies. Diamond-safe — fan-out and fan-in yield a single recomputation, never partial state.

familia: pureg.derived()

Effectus terminus

— effect · the world-touching

A side-effect sink. Where the graph reaches the world — logs, fetches, DOM, file IO. Always at the leaf, never in the middle.

familia: sinkg.effect()

On asking the graph why.

Most state libraries return a value when you read them. GraphReFly returns a value and a trail — every dependency it consulted, every transform it applied, every guard it passed through. The graph remembers how it knows what it knows.

One calls graph.describe({ explain }) and the library walks backwards through the dependency edges, returning a structured chain. Humans read it for confidence. Agents parse it for reasoning. Black-box decisions have no place in a system humans and machines share.

// A small pricing graph.
const g     = new Graph("pricing");
const price = g.state("price", 100);
const tax   = g.derived(
  "tax", [price],
  ([p]) => p * 0.1
);
const total = g.derived(
  "total", [price, tax],
  ([p, t]) => p + t
);

// — ask the graph why total is 110:
g.describe({ explain:
  { from: "price", to: "total" } });

// → { path: ["price", "tax", "total"],
//     transforms: ["× 0.1", "+"],
//     result: 110 }
