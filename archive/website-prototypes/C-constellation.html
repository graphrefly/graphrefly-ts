GraphReFly began as callbag-recharge -- a reactive streaming experiment. Over time it became clear that the real primitive wasn't streams, but a graph of interdependent values. Messages as tuples, two-phase push, observability without disturbing data flow. A protocol small enough to fit in your palm, powerful enough to let humans and LLMs co-operate on a single, inspectable artifact.

Core Principles

Built on fundamentals

One protocol

Messages are tuples -- [Type, Data?]. Sources, sinks, operators, graphs: all the same shape. Compose without adapters.

Diamond-safe

Two-phase push (DIRTY then DATA) ensures glitch-free propagation through any graph topology, including diamonds and cycles.

Graph as artifact

The graph is a first-class value. Describe it, snapshot it, serialize it, hand it to an LLM. Inspectable by design.

Human + LLM

Observe without disturbing. Both human developers and AI agents can read the graph state, reason about it, and participate -- without breaking data flow.

Applications

Where graphs fly

Orchestration

Coordinate multi-step workflows where each stage depends on upstream signals.

Message Queues

Back-pressure aware routing with tuple-based messages and composable operators.

AI Agentic Memory

Give LLM agents a shared, inspectable graph they can read and write without breaking flow.

Security

Audit every node, trace every message. The graph is the audit log.

Human + AI Ops

Humans set policy, AI executes. Both observe the same graph in real time.
