Every node checkpoints its state. Every derivation records its causal chain. When an LLM recommends an action, you can walk the graph backward and see exactly which inputs led to that conclusion.

Describe It, Don't Code It

NL-to-Graph Pipeline

Tell the system what you need in plain language. The LLM composes the reactive graph — nodes, edges, effects — and the runtime executes it. No boilerplate. No framework lock-in. Just intent to action.

How It Works

From intent to running system in three steps

一

Describe

Express your intent in natural language or Python. "Triage my inbox and flag anything from VIP senders about deadlines."

二

LLM Composes the Graph

The LLM translates your description into a reactive graph: source nodes for inputs, derived nodes for logic, effect nodes for actions.

三

Runs Reactively

The graph executes, checkpoints state on every change, and builds a causal trace. Inspect any decision. Replay from any checkpoint.

Five Lines to Reactive

State, derivation, effect — that's the whole model

example.py 
from graphrefly import state, derived, effect, DATA

count   = state(0)
doubled = derived([count], lambda c: c * 2)
effect([doubled], lambda d: print(d))

count.down([(DATA, 1)])  # prints: 2
